Answers

  1. What kind of jobs can I run on Hungry Hippos platform?

    You can run almost all types of jobs that you can run on Hadoop platform.

  2. What is the maximum size of data for which I can submit jobs for, using this tester web application?

    It really depends on RAM and CPU requirement of job/computation you are trying to perform. But as this application is made available for testing purposes right now, you can run jobs of maximum data size of 30 GB and computations that can be run on 2GB RAM on one node. This limitation is because when you submit a job, we create a 10 nodes cluster with each node having 2GB RAM and 40 GB hard disk.

    We have successfully tested computations on data of size of 10GB for sum and median metric calculations.

  3. Where should I put my data?

    Your data should be available and accessible over internet. You can use Amazon S3, Digital Ocean, UploadFiles or your own server accessible over internet.

  4. What all file formats are supported?

    As of now only CSV file format is supported.

  5. Can I submit multiple jobs in the one go?

    Yes. You can submit a job matrix which contains more than one job definitions.

  6. Which languages can I write job matrix implementation in?

    As of now, only 'Java' is supported.

  7. What code do I need to write for job matrix?

    You need to provide your own implementation of com.talentica.hungryHippos.client.job.JobMatrix interface. For more details on client API please refer javadocs here

    Please download the current version of hungry hippos client JAR from here

    You can also refer to few sample job matrix implementations from here.
